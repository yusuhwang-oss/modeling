# v2. 업종 평균 대비 상대지표 적용 — 모델 개선

v1 모델을 실전 검증(삼성전자·현대차·SK하이닉스·SOOP·빙그레)하는 과정에서, **SK하이닉스처럼 업황이 급격히
나빠진 시기의 우량기업을 분식기업과 혼동하는 현상**을 발견했습니다. 이를 개선하기 위해 8개 재무비율을
"업종 평균 대비 상대값"으로 변환하는 작업을 진행했습니다.

## 문제 정의

v1 모델은 재무비율의 **절대값**만 보고 판단했습니다. 그런데 2022년처럼 특정 업종 전체가 어려웠던 시기에는,
정상기업도 재무비율이 나빠질 수 있습니다. 모델이 "이 회사가 원래 안 좋은 건지, 업종 전체가 안 좋은 건지"를
구분하지 못하는 것이 문제였습니다.

## 개선 방법

```
상대지표 = 원래 재무비율 / 같은 업종 정상기업들의 평균 재무비율
```

값이 1보다 크면 업종 평균보다 높다는 뜻이고, 1보다 작으면 업종 평균보다 낮다는 뜻입니다.

### 업종 평균의 3단계 기준선 (표본 부족 문제 대응)

DART의 업종코드는 매우 세분화되어 있어, 87개 표본을 그대로 나누면 업종당 정상기업이 1개뿐인 경우가
대부분(64개 업종 중 47개)이었습니다. 이를 해결하기 위해 3단계 기준선을 적용했습니다.

| 우선순위 | 기준 | 조건 |
|---|---|---|
| 1순위 | 정확히 같은 업종코드 평균 | 해당 업종 정상기업 3개 이상 |
| 2순위 | 업종코드 앞 3자리(업종군) 평균 | 업종군 내 정상기업 3개 이상 |
| 3순위 | 전체 정상기업 평균 | 위 조건을 모두 만족하지 못할 때 |

## 표본 보강

- 11개 분식기업이 속한 업종 + SK하이닉스가 속한 업종(반도체 제조업, 2612)을 타겟으로 정상기업을 추가 수집
- 무작위 추출(400개 후보 조회) + 알려진 반도체 기업 직접 확인(DB하이텍, 하나마이크론, LX세미콘 등)을 병행
- 최종 표본: 87개 → **97개** (분식 11 + 정상 86)

## 결과 비교

| 회사 | v1 (업종 미반영) | v2 (업종 반영) | 실제 상태 |
|---|---|---|---|
| 삼성전자 | 12.9% | 4.2% | 정상 |
| 현대자동차 | 9.9% | 21.9% | 정상 |
| **SK하이닉스** | **34.9%** (SOOP보다 높음) | **15.2%** (SOOP보다 낮음) | 정상 |
| 빙그레 | 13.6% | 15.2% | 정상 |
| SOOP | 32.3% | **26.7%** (5개 중 최고) | 분식 확정 |

**가장 중요한 성과**: v1에서는 SK하이닉스(정상)가 SOOP(분식 확정)보다 높은 확률로 잘못 판정되었는데,
v2에서는 이 순서가 정상화되어 **SOOP이 5개 회사 중 가장 높은 확률**을 기록했습니다.

## 한계 및 향후 개선 방향

- 표본이 97개로 여전히 작아, 3단계 기준선 적용에도 불구하고 "3순위(전체 평균)"로 처리된 표본이 48개
  (전체의 절반)에 달합니다.
- 업종 코드가 정교하게 매칭되지 않는 회사(예: 지주회사로 분류된 대기업)는 업종 보정의 효과를 충분히
  누리지 못합니다.
- 딥러닝 모델 특성상 실행할 때마다 결과가 소폭 달라지는 재현성 문제는 v2에서도 완전히 해결되지 않았습니다.

## 사용 방법

`fraud_prediction_model_v2.ipynb`와 `training_data_final_v2.csv`를 같은 폴더에 두고 노트북을 순서대로
실행하면 됩니다. `scripts/` 폴더에는 업종코드 조회부터 표본 보강까지의 전체 개발 과정이 번호 순서대로
정리되어 있습니다 (17~24번, v1의 01~16번 스크립트에 이어지는 번호입니다).
